Jak zacząłem przygodę z programowaniem

15 marca 2025
Programowanie Scratch Gry Nauka

Każda pasja ma swój początek. Moja przygoda z programowaniem zaczęła się, gdy miałem 10 lat i odkryłem Scratch - wizualny język programowania stworzony specjalnie dla dzieci. To, co zaczęło się jako zabawa, przekształciło się w prawdziwą pasję, która trwa do dziś.

Pierwsze kroki w świecie kodu

Wszystko zaczęło się, gdy mój nauczyciel informatyki pokazał nam Scratch podczas lekcji. Byłem zafascynowany tym, jak łatwo można było tworzyć proste gry i animacje, po prostu łącząc kolorowe bloczki. To było jak układanie klocków LEGO, ale w wirtualnym świecie, gdzie efekty naszej pracy ożywały na ekranie.

Po kilku próbach w szkole, nie mogłem się doczekać, aby kontynuować naukę w domu. Poprosiłem rodziców o możliwość korzystania z komputera po szkole i zacząłem eksperymentować. Początkowo tworzyłem bardzo proste projekty - animowane postacie, które reagowały na kliknięcia myszką, albo proste quizy.

Moja pierwsza gra - "Pixel Adventure"

Po kilku tygodniach eksperymentowania postanowiłem stworzyć moją pierwszą "prawdziwą" grę. Nazwałem ją "Pixel Adventure" (tak, już wtedy lubiłem pikselową grafikę!). Była to prosta platformówka, w której sterowany przez gracza kot musiał zbierać monety i unikać przeszkód.

Gra była bardzo prosta, ale dla 10-letniego mnie była to ogromna rzecz! Pamiętam ekscytację, gdy po raz pierwszy udało mi się zaimplementować detekcję kolizji, aby kot "umierał" po dotknięciu wroga. Byłem dumny, gdy udało mi się dodać licznik punktów i system "żyć".

Wyzwania i rozwiązania

Najważniejszą lekcją, jaką wyniosłem z tego pierwszego projektu, było to, że programowanie to nie tylko pisanie kodu, ale przede wszystkim rozwiązywanie problemów. Każdy nowy element gry przynosił nowe wyzwania:

  • Jak sprawić, by kot nie mógł przechodzić przez platformy?
  • Jak zaimplementować grawitację, żeby skok wyglądał realistycznie?
  • Jak stworzyć poruszających się wrogów z różnymi wzorcami ruchu?
  • Jak zaprojektować różne poziomy gry?

Każdy rozwiązany problem dawał mi ogromną satysfakcję. Zacząłem rozumieć, że programowanie to jak rozwiązywanie zagadek - trzeba myśleć logicznie i krok po kroku dochodzić do rozwiązania.

Od zabawy do pasji

To, co zaczęło się jako zabawa w Scratch, szybko stało się moją pasją. Spędzałem godziny na doskonaleniu mojej gry, dodawaniu nowych funkcji i poziomów. Pokazałem ją kolegom z klasy, co dało mi dodatkową motywację - ich zachwyt i sugestie były dla mnie bardzo cenne.

Najważniejsze, czego się nauczyłem, to fakt, że komputer zrobi dokładnie to, co mu każesz - nie więcej i nie mniej. To była moja pierwsza lekcja logicznego myślenia i precyzyjnego formułowania instrukcji.

Po sukcesie mojej pierwszej gry zacząłem tworzyć kolejne projekty w Scratch. Z czasem zaczęło mi brakować możliwości i chciałem więcej. To był moment, gdy zacząłem interesować się HTML i CSS, aby tworzyć strony internetowe, a później JavaScript, aby dodawać do nich interaktywność.

Co dał mi Scratch?

Patrząc z dzisiejszej perspektywy, Scratch dał mi solidne podstawy do nauki "prawdziwego" programowania:

  • Zrozumienie podstawowych koncepcji, takich jak zmienne, pętle i instrukcje warunkowe
  • Myślenie algorytmiczne i rozwiązywanie problemów
  • Planowanie i organizację kodu
  • Debugowanie - umiejętność znajdowania i naprawiania błędów
  • Podstawy projektowania gier i interakcji użytkownika

Te umiejętności okazały się bezcenne, gdy później zacząłem uczyć się bardziej zaawansowanych języków programowania. Koncepcje były podobne, zmieniała się tylko składnia i możliwości.

Mój pierwszy kod w JavaScript

Dla porównania, oto jak wyglądał mój pierwszy kod w JavaScript, kiedy zacząłem uczyć się tego języka rok później: 


// Prosty quiz o programowaniu
let punkty = 0;
const liczbaPytan = 3;

// Pytanie 1
let odpowiedz1 = prompt("Jaki język programowania służy do tworzenia stron internetowych? (HTML, Python, czy C++)");
if (odpowiedz1.toLowerCase() === "html") {
    alert("Dobrze!");
    punkty++;
} else {
    alert("Niestety nie. Poprawna odpowiedź to HTML.");
}

// Pytanie 2
let odpowiedz2 = prompt("Co to jest zmienna w programowaniu?");
if (odpowiedz2.toLowerCase().includes("przechowuje") || odpowiedz2.toLowerCase().includes("wartość")) {
    alert("Dobrze!");
    punkty++;
} else {
    alert("Niestety nie. Zmienna to miejsce do przechowywania wartości.");
}

// Pytanie 3
let odpowiedz3 = prompt("Co oznacza skrót CSS?");
if (odpowiedz3.toLowerCase() === "cascading style sheets") {
    alert("Dobrze!");
    punkty++;
} else {
    alert("Niestety nie. CSS to Cascading Style Sheets.");
}

// Wynik
alert("Zdobyłeś " + punkty + " punktów na " + liczbaPytan + " możliwych!");
if (punkty === liczbaPytan) {
    alert("Gratulacje! Perfekcyjny wynik!");
} else if (punkty >= liczbaPytan / 2) {
    alert("Całkiem nieźle! Spróbuj jeszcze raz, aby poprawić wynik.");
} else {
    alert("Spróbuj jeszcze raz, aby poprawić swój wynik.");
}

Widać tutaj, jak koncepcje, które poznałem w Scratch (zmienne, instrukcje warunkowe, interakcja z użytkownikiem), przekładają się na "prawdziwy" kod. Różnica polega głównie na tym, że zamiast przeciągać bloczki, musiałem napisać wszystko tekstowo.

Porady dla początkujących

Jeśli jesteś początkującym programistą, mam dla Ciebie kilka rad na podstawie mojego doświadczenia:

  1. Zacznij od prostych narzędzi - Scratch, Blockly czy inne wizualne języki programowania są świetnym początkiem
  2. Twórz projekty, które Cię interesują - motywacja jest kluczowa!
  3. Nie bój się popełniać błędów - to normalna część procesu nauki
  4. Dziel się swoimi projektami z innymi - feedback jest bezcenny
  5. Bądź cierpliwy - nauka programowania to maraton, nie sprint

Podsumowanie

Moja przygoda z programowaniem zaczęła się od prostej gry w Scratch, ale doprowadziła mnie do miejsca, w którym jestem teraz - tworzę własne strony internetowe, eksperymentuję z AI i wciąż uczę się nowych technologii. Najważniejsze, że cały czas sprawia mi to ogromną radość!
